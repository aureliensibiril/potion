## Activation

```bash

claude plugin add ./potion

# Or for development/testing:
claude --plugin-dir ./potion
```

## Usage examples

```
# Ask about the codebase
/potion:ask Where is the auth middleware?
/potion:ask How does the billing module work?

# Plan before implementing
/potion:plan Add user preferences with a new settings page
/potion:plan Refactor the billing module to support multiple providers

# Implement features
/potion:implement Add a new REST endpoint for user preferences
/potion:implement Create a background job for sending notifications

# Review code
/potion:review Review my changes to the payment flow
/potion:review Check the new migration for issues
```

Edit `guidelines.md` to adjust the codebase knowledge that all skills reference.
Sections wrapped in `<!-- user-edited -->` markers will be preserved if you
re-generate the skill pack using refresh mode.

## Refreshing

When your codebase changes, re-run the generator in refresh mode:

```

```

This re-analyzes changed modules, updates guidelines, and regenerates affected
skills — preserving your `<!-- user-edited -->` sections.
